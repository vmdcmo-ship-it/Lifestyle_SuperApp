# Life Insurance UI/UX Guide
## Partnership with Cathay Life - White-Label Distribution

> **3 Naming Vibes for Maximum Conversion**

---

## 🎯 Product Naming Philosophy

### Core Strategy: "Hide Partner, Show Value"

**UX Journey:**
```
① Touch Pain Point (Nhu cầu)
   ↓
② Show Solution (Giải pháp)
   ↓
③ Build Trust (Uy tín đối tác - Bottom of page)
```

**Why this works:**
- Customer focuses on **their needs**, not brand names
- Lifestyle branding = Familiar, trustworthy
- Cathay Life badge = Credibility booster (after interest)

## 📱 Screen Architecture

### App Structure

**User App:**
```
Lifestyle Tab
├─ 💰 Ví Lifestyle
├─ 🎁 Tiết kiệm
├─ 🛡️ Bảo hiểm
│   ├─ Xe (TNDS, Vật chất)
│   ├─ Sức khỏe (BHXH)
│   └─ 🌟 Bản Thiết Kế Tương Lai ← NEW (Life Insurance)
│       ├─ Danh sách sản phẩm
│       ├─ 📊 Bảo hiểm của tôi (Timeline)
│       └─ 📚 Wiki Bảo Hiểm
└─ 🎫 Voucher
```

**Driver App:**
```
Tài xế (Profile) Tab
├─ 💰 Thu nhập
├─ 🛡️ Bảo hiểm
│   ├─ TNDS xe
│   ├─ Của để dành (BHXH)
│   └─ 🛡️ Lá Chắn Tài Chính ← NEW (Life Insurance)
│       ├─ Sản phẩm phù hợp
│       └─ Bảo hiểm của tôi
└─ ⚙️ Cài đặt
```

**Merchant App:**
```
Thêm (More) Tab
├─ 💰 Tài chính

├─ 🛡️ Bảo hiểm
│   ├─ Xe giao hàng
│   ├─ Phúc lợi chủ hộ (BHXH)
```

## 📊 Conversion Optimization Strategies

### 1. Vibe Selection (A/B Test)

**Default vibe by user segment:**

```typescript
function getDefaultVibe(user: User): ProductNamingVibe {
  // Young professionals → LIFESTYLE
  if (user.age >= 25 && user.age <= 40 && !user.isBusinessOwner) {
    return ProductNamingVibe.LIFESTYLE;
  }
  
  // Business owners, merchants, drivers → FINANCE
  if (user.isBusinessOwner || user.role === 'DRIVER' || user.role === 'MERCHANT') {
    return ProductNamingVibe.FINANCE;
  }
  
  // Mass market, traditional → TRUST
  return ProductNamingVibe.TRUST;
}
```

**User can switch vibe:**
- Tab switcher at top: [Cuộc đời] [Tài chính] [Của để dành]
- Track which vibe converts best per segment

### 2. Progressive Disclosure

**Product page scroll depth tracking:**

```
Section 1 (Pain Point): 80% scroll
Section 2 (Solution): 60% scroll
Section 3 (Partner Trust): 30% scroll

→ Optimize content length based on data
```

**CTA placement:**
- Above the fold (after pain point): Primary CTA
- After benefits: Secondary CTA
- Bottom (after partner): Tertiary CTA

**Hypothesis**: Users who scroll to partner section have 2x conversion rate

### 3. Social Proof Placement

**Strategic placement:**
- Video testimonial: After benefits, before comparison
- Star ratings: Top of page (near product name)
- "X customers bought this": Urgency badge

**Types:**
- Video (most powerful)
- Text testimonials with photo
- Stats ("12,345 families protected")
- Trust badges (Cathay Life rating)

### 4. Comparison Tables

**Format:**
```
Gửi NH vs BH Đầu tư
Tự bảo vệ vs Mua BH
Không BH vs Có BH
```

**Psychology:** Make the "buy insurance" column always WIN visually

### 5. Price Anchoring

**Show total cost breakdown:**
```
₫2,250,000/tháng
= ₫75,000/ngày
= ₫3,125/giờ
= 2.5 bát phở mỗi ngày ← Relatable!
```

**Compare to alternatives:**
```
Học phí đại học: 30M/năm
Chi phí bảo hiểm: 24M/năm
→ Chỉ bằng 80% học phí, nhưng bảo vệ trọn đời!
```

### 6. Timeline Dashboard Engagement

**Gamification elements:**
- Progress bars (Visual satisfaction)
- Milestones ("Bạn đã hoàn thành 5 năm! 🎉")
- Badges ("Khách hàng trung thành 10 năm 🏆")

**Retention hooks:**
- Next payment countdown (Urgency)
- Cash value growth chart (Visual progress)
- Projected pension at retirement (Goal visualization)
